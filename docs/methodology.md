# Methodology

## 1. Baseline lock (ทำซ้ำได้)
บันทึกใน `data/baseline_lock.json`:
- ชื่อไฟล์ + SHA-256 + ขนาด (bytes)
- จำนวนหน้า + metadata (creator=Canva ฯลฯ)
- เวอร์ชัน Python + dependencies + แพลตฟอร์ม
- วันเวลา (UTC)

SHA-256 ต้นฉบับ: `d304742f0d8e0971fa93affcf2beb43e6fef5e7d848f6db6e322e57aafb1b9c6`

## 2. ทำไมต้องตรวจจากภาพ ไม่ใช่ text
ฟอนต์ใน PDF เป็น Type0 subset (Identity-H) ที่ ToUnicode เสีย → `page.get_text()` คืนค่า mojibake

## 3. การตรวจรายหน้า (Round 1)
1. render ทุกหน้า 150 DPI (`01_extract_pdf.py`) → `data/page_images/`
2. ตรวจเต็มหน้าทุกหน้า (1–72) หา: สระ/วรรณยุกต์ผิดตำแหน่ง, อักขระซ้อน, อักขระหาย,
   เว้นวรรคผิด, คำแตก/คำติด, ข้อความล้น/ตัด, ความผิดในตาราง/หัวข้อ/เลขหน้า/caption
3. จุดที่น่าสงสัยหรือสำคัญ → ซูม 300–600% (`data/crops/`) เพื่อยืนยัน
4. ยืนยันซ้ำด้วยหลักฐานที่ 2: วิเคราะห์ฟอนต์รายสแปน (`get_text("dict")`) และเทียบคำเดียวกันต่างหน้า

## 4. เกณฑ์ยืนยัน
- **Confirmed** = มีหลักฐาน ≥ 2 สัญญาณ (เช่น ซูมเห็นชัด + เทียบคำถูกจากหน้าอื่น/สารบัญ)
- **Human Review Required** = ยืนยันไม่ได้ 100% → ไม่นับเป็น 0

## 5. บทเรียนสำคัญจากรอบ 1
TH-001 (หัวเรื่องหน้า 66) **หลุด** จากการตรวจเต็มหน้า 150 DPI จับได้เฉพาะตอนซูม →
สรุปว่า การตรวจเต็มหน้าอย่างเดียว **ไม่พอ** สำหรับอักขระเพี้ยนระดับกลิฟ
จึงกำหนดให้ **รอบ 2 = ซูมหัวข้อ/แถบสีทุกจุด** และแนะนำพิสูจน์อักษรโดยมนุษย์

## 6. การตรวจหลังแก้ (รอบถัดไป)
รับ PDF ที่ export จาก Canva → render ใหม่ → เทียบภาพหน้าเดิม↔ใหม่ (`04_render_and_compare.py`) →
